市场上常见的关系型数据库有mysql 、SQL Server、Oracle、Sybase、DB2、Microsoft Access等。

1. MySQL
MySQL是一个关系型数据库管理系统，也是最流行的关系型数据库管理系统之一。它将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。在 WEB 应用方面，它也是最好的关系数据库管理系统)应用软件之一。优点体现在体积小、速度快、总体拥有成本低，尤其是使用开放源码，这使得它可以适用于一般中小型网站的开发。

2. Microsoft SQL Server
由Microsoft 公司推出，具有使用方便可伸缩性好与相关软件集成程度高等优点，可跨越多种大型多处理器的服务器等多种平台使用，可以用于构建和管理用于业务的高可用和高性能的数据应用程序，主机操作系统为window，主要还是用于web网站的建设，承载中小型web后台数据。在租赁的虚拟主机中一般会预安装它作为数据库软件。但只能运行在微软的windows平台，没有开放性可言，并行实施和共存模型并不成熟，很难处理日益增多的用户数和数据卷，伸缩性有限，当用户连接多时性能会变的很差，并且不够稳定，目前只支持C/S模式。

3. Oracle
Oracle是全世界第一个开放式商品化关系型数据库管理系统，能够支持多种平台，采用标准的SQL结构化查询语言，支持多种数据类型，提供面向对象存储的数据支持，具有第四代语言开发工具，如Unix、Windows NT、OS/2、Novell，并且具有很好的并行处理功能，Oracle产品主要由Oracle服务器产品、开发工具、应用软件组成，也有基于微机的数据库产品，所以较为适用于银行、金融、保险等行业的大型企业开发大型数据库。

4. DB2
DB2是美国IBM公司开发的一套关系型数据库管理系统。具有较好的可伸缩性，可支持从大型机到单用户环境，应用于所有常见的服务器操作系统平台下。 DB2提供了高层次的数据利用性、完整性、安全性、可恢复性，以及小规模到大规模应用程序的执行能力，具有与平台无关的基本功能和SQL命令。它采用了数据分级技术，能够使大型机数据很方便地下载到LAN数据库服务器，使得客户机/服务器用户和基于LAN的应用程序可以访问大型机数据，并使数据库本地化及远程连接透明化。并且DB2拥有一个非常完备的查询优化器而著称，其外部连接改善了查询性能，并支持多任务并行查询。 它还具有很好的网络支持能力，每个子系统可以连接十几万个分布式用户，可同时激活上千个活动线程，所以它非常适用于大型分布式应用系统的相关应用。

5. Microsoft Access
Access是由微软发布的关系数据库管理系统，是 Microsoft Office 的系统程序之一。它数据库引擎的图形用户界面和软件开发工具结合在一起，所以是一种小型数据库、桌面数据库，容易操作，使用广，成本低，存储方式单一，但记录数过多，一般记录数达到10万条左右的时候性能就会急剧下降，数据储存量小安全性不够高，加了用户级密码容易破解，它适合数据量少的应用，在处理少量数据和单机访问的数据库时效果较好，效率也较高
